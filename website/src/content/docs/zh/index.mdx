## 为什么选择 Pareto

如果你使用过 Next.js 或 Remix，你已经熟悉这些模式了。Pareto 使用相同的约定，但去除了复杂性。

### 无 Server Components

只有普通的 React 组件加上数据 loader。没有服务端和客户端之间的心智模型割裂。

### 无框架锁定

标准的 Express 服务器，标准的 Vite 构建。你的代码保持可移植性。

### 开箱即用的流式渲染

`defer()` 和 `<Await>` 通过 Suspense 边界实现渐进式数据加载 — 不再有全页面的加载动画。

### 内置状态管理

`defineStore()` 配合 Immer 变更和自动的服务端/客户端序列化。无需额外依赖。

### 基于文件的路由

`page.tsx`、`layout.tsx`、`head.tsx`、`loader.ts` — 你已经熟悉的约定。
